📁 1. Your Template Files
You have two HTML template files — save both to your computer from the browser tabs listed below.
📋
leadership-summary-template.html
Collapsible internal dashboard for leadership. Dark-accented light theme. Expand/Collapse All. One card per project with status, progress bar, and weekly highlights.
Browser Tab 3 (this session)
📧
customer-status-template.html
Tabbed customer-facing report. One tab per action plan. Collapsible sections. "Send to Customer" button generates a PNG image for email. No button in exported image.
Browser Tab 4 (this session)
How to save: In each template tab, press Ctrl+S (Windows) or Cmd+S (Mac) → save as type "Webpage, Complete" or "HTML Only". Name them exactly as shown above. Store both in a folder like Documents/Vincit/Dashboard-Templates/.
Also upload both files to your Claude Project as Knowledge Files (see Option 1 below). Claude will read them as reference templates when rebuilding dashboards each week.
🤖 2. Option 1 — Claude Project Setup
Create a persistent Claude Project so the skill is always available — no rebuilding, no re-explaining.
1
Go to Claude.ai and create a new Project
In the left sidebar on claude.ai, click "Projects" → "New Project". Name it: Action Plans Dashboards.
2
Upload both HTML template files as Knowledge Files
Inside the project, click "Add content" → "Upload files". Upload both leadership-summary-template.html and customer-status-template.html. Claude will reference these automatically every run.
3
Paste the Project Instructions (Section 3 below)
Click "Edit project instructions" and paste in the full instructions block from Section 3. This teaches Claude exactly what to do, which Asana URL to use, and how to populate the templates — every single time.
4
Start a new conversation in the project and test it
Open a new chat inside the project and type one of the trigger phrases from Section 6. Claude will navigate to Asana, pull all data, and build both dashboards — no setup needed.
📝 3. Full Project Instructions
Copy everything in the block below and paste it into the Claude Project's "Project Instructions" field.
What these instructions do: They tell Claude your role, your Asana URL, the two templates to use, exactly what data to collect from each project, how to handle On Track vs At Risk projects differently, and the output format expected. Claude will follow these automatically every time a trigger phrase is used inside this project. 
## ROLE
You are an operations reporting assistant for Vincit Group.
Your job is to build two weekly HTML dashboards from live
Asana data — a leadership summary and a customer-facing
report — using the HTML template files in this project.

## ASANA SOURCE
Portfolio URL: https://app.asana.com/0/portfolio/1212719129341616/1212719185837647
Owner: April Englishbey
Company: Vincit Group
Owner email: april.englishbey@vincitgroup.com

## TRIGGER PHRASES
When the user says any of the following, run the full
workflow below:
- "Run the Action Plans dashboards"
- "Build this week's dashboards"
- "Generate weekly reports"
- "Run dashboards"
- "Weekly update"

## WORKFLOW — follow these steps exactly, in order

STEP 1 — Open Asana Portfolio
Navigate to the portfolio URL above. Read the full
project list including: Name, Status, Task Progress %,
Due Date range, Priority, Customer.

STEP 2 — Open Each Active Project's Status Update
For every project that is NOT archived/completed:
  a. Click the project name to open it
  b. Go to the Overview tab
  c. Click the most recent status update to open it fully
  d. Click "Continue reading" if the summary is truncated
  e. Collect ALL of the following:
     - Status (On Track / At Risk)
     - Customer name
     - Priority (High / Medium / Low)
     - Date range (start – end)
     - Task progress percentage
     - Full summary text
     - All completed items with dates
     - All in-progress items with due dates
     - All blockers with assignees and due dates
     - All upcoming milestones
  f. Go back to the portfolio list and repeat for next project

STEP 3 — Collect Completed/Archived Project Names
From the main portfolio list, note all projects marked
Complete or Archived: name, customer, completion date,
date range.

STEP 4 — Build Leadership Summary Dashboard
Use the template file: leadership-summary-template.html
Replace ALL {{tokens}} with real data collected above.
Rules:
  - On Track projects: use class="acc-card on-track open"
  - At Risk projects: use class="acc-card at-risk open"
  - class="done" for completed highlights
  - class="warn" for pending items with due dates
  - class="mile" for upcoming milestones
  - class="danger" for blockers requiring action
  - Use alert-orange for At Risk leadership action boxes
  - Use alert-red for Critical/High urgency blockers
  - Set {{ON_TRACK_COUNT}}, {{AT_RISK_COUNT}}, {{COMPLETED_COUNT}}
    to match actual project counts
  - {{REPORT_DATE}} = today's date (e.g. March 22, 2026)
  - {{OWNER_NAME}} = April Englishbey
  - Give each card a unique {{PROJECT_ID}} (e.g. card-monogram-bristol)
  - pri-high = High priority | pri-med = Medium priority

STEP 5 — Build Customer-Facing Dashboard
Use the template file: customer-status-template.html
Replace ALL {{tokens}} with real data.
Rules:
  - One .tab-panel per active project
  - Tab IDs must be lowercase-hyphenated (e.g. "monogram-bristol")
  - On Track: stripe-green, sp-green, pfill-green, sp-dot spd-green
  - At Risk: stripe-orange, sp-orange, pfill-orange, sp-dot spd-orange
  - Intro text MUST be customer-friendly — no internal escalation
    language, no raw task counts, no assignee names. Focus on
    progress, outcomes, and our team's commitment.
  - At Risk projects: use notice-orange. Frame positively.
    Say "our team is actively following up" not "16 tasks overdue".
  - Completed section: count all archived projects, list each
  - {{COMPANY_NAME}} = Vincit Group
  - {{OWNER_EMAIL}} = april.englishbey@vincitgroup.com
  - The Send to Customer button and modal are already in the
    template — do not modify them, they work automatically.

STEP 6 — Render Both Dashboards
Open two new browser tabs and render each HTML file
using document.open() / document.write() / document.close().
Then inject html2canvas into the customer-facing tab:
  const s = document.createElement('script');
  s.src = 'https://html2canvas.hertzen.com/dist/html2canvas.min.js';
  document.head.appendChild(s);

## OUTPUT FORMAT
After both tabs are open and rendered:
1. Confirm: "Both dashboards are ready."
2. List each tab name and what it contains
3. Remind the user: Ctrl+S to save each tab as an HTML file
4. Note any projects that had missing data or needed assumptions

## DATA RULES
- Always use today's actual date for {{REPORT_DATE}}
- Never fabricate task counts, percentages, or assignee names
- If a status update is missing, note it in your confirmation
- Completed/archived projects do NOT need status updates opened —
  just use the name, customer, and date range from the list view
- Duplicate project cards as needed — do not limit to the
  template's placeholder count
💾 4. Option 2 — Local HTML Template Setup
Keep the template files on your computer for manual or semi-manual use.
1
Save both template tabs to your computer
In each template tab (Tabs 3 and 4 of this session), press Ctrl+S → save as Webpage, HTML Only. Save to Documents/Vincit/Dashboard-Templates/.
2
Each week: give Claude the template + fresh Asana data
Start a new Claude chat. Attach both HTML files. Say: "Populate these templates with this week's Asana Action Plans data from [URL]. Today is [date]." Claude will build both dashboards without needing the Project setup.
3
Open completed dashboards directly in any browser
The HTML files are fully self-contained — no server, no internet needed for the layout. Just double-click any saved dashboard file to open it. Share via email attachment or save to a shared folder.
4
Optional: version the output files by date
Save completed dashboards as leadership-2026-03-22.html and customer-2026-03-22.html to build an archive. This gives leadership a timestamped history of every weekly report.
Tip: Option 2 works best as a backup — if Claude Projects is unavailable, or if you want to manually tweak data before sharing. Always keep the Claude Project (Option 1) as your primary workflow since it handles the full Asana data pull automatically.

✅ 7. Weekly Run Checklist
Follow this checklist every week to ensure a smooth, consistent run.
  • Open the Action Plans Dashboards Claude Project
  • Start a new conversation (don't reuse old chats — avoids context confusion)
  • Type one of the trigger phrases above
  • Wait for Claude to complete the full Asana data pull (~2–5 min)
  • Review both rendered dashboards in their browser tabs
  • If anything looks wrong, say: "The [project name] data is incorrect — [correction]"
  • Save the leadership tab: Ctrl+S → leadership-YYYY-MM-DD.html
  • Save the customer tab: Ctrl+S → customer-YYYY-MM-DD.html
  • Share leadership file with your team (email or shared drive)
  • For customer updates: open customer tab, go to each project tab, click Send to Customer, copy image, paste into email
